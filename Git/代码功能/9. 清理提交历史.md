### 1. 多个提交合并成一个提交

- **方法一：rebase**

   a. 打开交互式变基界面

   ```bash
   git rebase -i <目标版本>
   ```

  - 合并所有历史

   ```bash
   git rebase -i --root
   ```

   b. 编辑提交列表

   c. 编辑合并后的提交信息

   d. 如果变基涉及的提交已经同步至远程仓库：

   ```bash
   git push <仓库名> <分支名> --force-with-lease
   ```

- **方法二：reset**

   a. 软重置到目标提交：

   ```bash
   git reset --soft <目标版本>
   ```

   b. 创建新的提交

### 2. 当前工作区内容合并到上一个提交

- **方法一：`git commit --amend`**

   a. 确保你的工作区中没有未跟踪的文件，或者你已经将它们添加到暂存区:

   ```bash
   git add .
   ```

   b. 将当前工作区的更改合并到上一个提交中：

   ```bash
   git commit --amend
   ```

   c. 编辑提交信息：

   d. 如果已经将上一个提交推送到远程仓库:

   ```bash
   git push --force-with-lease
   ```

- **方法二：使用 `git reset`**

   a. 将当前分支重置到上一个提交，但保留暂存区和工作区的更改。

   ```bash
   git reset HEAD~ --soft
   ```

   c. 提交。

   d. 如果已经将上一个提交推送到远程仓库:

   ```bash
   git push --force-with-lease
   ```
